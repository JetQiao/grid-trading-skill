---

## What This Skill Does

Builds, simulates, and analyses grid trading strategies for any asset.
Given a price range, capital, and grid count, it:

1. Constructs either an **arithmetic** (equal-spacing) or **geometric**
   (equal-ratio) grid of buy/sell levels.
2. Runs an **event-driven backtest** over historical or synthetic price data.
3. Tracks **position & PnL** in real time (buy cost basis, realized/unrealized
   gains, fees).
4. Enforces **risk rules** (capital floor, drawdown limit, stop-loss,
   take-profit).
5. Outputs a **performance report** and **per-level grid table**.

## Input Format

### Minimal (text)

```
symbol=BTC/USDT, lower=40000, upper=60000, grids=20,
capital=10000, type=geometric, fee=0.001
```

### Full GridConfig (Python)

```python
from grid_trading.strategy.grid_strategy import GridConfig, GridStrategy

config = GridConfig(
    symbol        = "BTC/USDT",
    grid_type     = "arithmetic",   # or "geometric"
    price_lower   = 40000,
    price_upper   = 60000,
    grid_count    = 20,
    total_capital = 10000,
    fee_rate      = 0.001,           # 0.1% per side
    stop_loss_price   = 38000,       # optional
    take_profit_price = 62000,       # optional
)
```

### Backtest with price data

```python
from grid_trading.backtest.simulator import BacktestSimulator
from grid_trading.tests.mock_data import sine_wave   # or load CSV

price_series = sine_wave(base_price=50000, amplitude=5000)
# or: [(timestamp_float, price_float), ...]

sim = BacktestSimulator(GridStrategy(config))
result = sim.run(price_series)
sim.print_report(result)
```

### 4. Risk Warnings

Risk alerts are structured as:

```python
RiskAlert(
    level            = "critical",          # or "warning"
    type             = "below_lower",       # see list below
    message          = "Price 38500 is below grid lower bound 40000. "
                       "Unrealized loss = 22.1% of capital.",
    suggested_action = "Stop adding positions. Consider rebalancing …",
)
```

Alert types:

| Type                 | Level    | Trigger                                         |
|----------------------|----------|-------------------------------------------------|
| `stop_loss_breach`   | critical | price ≤ stop_loss_price                         |
| `take_profit_reached`| warning  | price ≥ take_profit_price                       |
| `below_lower`        | warning / critical | price < price_lower (critical if loss > 20%) |
| `above_upper`        | warning  | price > price_upper                             |
| `capital_low`        | warning  | available capital < 10% of total                |
| `drawdown_limit`     | critical | unrealized loss > 20% of total capital          |

## Quick-Start Example

```python
from grid_trading.strategy.grid_strategy import GridConfig, GridStrategy
from grid_trading.backtest.simulator import BacktestSimulator
from grid_trading.tests.mock_data import sine_wave

# 1. Configure
config = GridConfig(
    symbol="BTC/USDT", grid_type="arithmetic",
    price_lower=44000, price_upper=56000,
    grid_count=12, total_capital=10000, fee_rate=0.001,
)

# 2. Run backtest
sim = BacktestSimulator(GridStrategy(config))
result = sim.run(sine_wave(base_price=50000, amplitude=5000, periods=5))
sim.print_report(result)

# 3. Inspect grid table
from grid_trading.core.grid_builder import GridBuilder
builder = GridBuilder(fee_rate=0.001)
for lvl in builder.build_arithmetic(44000, 56000, 12, 10000):
    print(f"L{lvl.level:2d} buy={lvl.buy_price:.2f} sell={lvl.sell_price:.2f} "
          f"qty={lvl.quantity:.6f} profit={lvl.expected_profit:.4f}")
```

## Design Constraints

- No exchange SDK dependency — all price data via `list[tuple[float, float]]`.
- 8-decimal precision on all prices (crypto-grade).
- All percentages stored as decimals internally; converted for display.
- No global state — every run is isolated to its `GridStrategy` instance.
- Idempotency: only one PENDING order per `(level, side)` at any time.
- Grid spacing must satisfy `step_ratio > 2 × fee_rate`; `GridBuilder`
  raises `ValueError` otherwise and `recommend_grid_count()` gives a safe bound.
